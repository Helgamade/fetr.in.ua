# Настройка скриптов управления сервером

## Установка прав на выполнение

Если скрипты не запускаются (ошибка "Permission denied"), установите права:

```bash
cd /home/idesig02/fetr.in.ua/www
chmod +x server/*.sh
```

## Остановка процесса, запущенного через админ панель

Если сервер запущен через админ панель хостинга, его нужно остановить перед запуском через SSH:

```bash
# 1. Найти процесс
ps aux | grep "node.*server/index.js"

# 2. Остановить по PID (замените 3930503 на реальный PID)
kill 3930503

# Или остановить все процессы node, связанные с server/index.js
pkill -f "node.*server/index.js"

# 3. Проверить, что процесс остановлен
ps aux | grep "node.*server/index.js"
```

## Запуск через SSH

После остановки процесса из админ панели:

```bash
cd /home/idesig02/fetr.in.ua/www
./server/start.sh
```

## Проверка статуса

```bash
./server/status.sh
```

## Остановка через SSH

```bash
./server/stop.sh
```

## Перезапуск

```bash
./server/restart.sh
```

## Важные замечания

1. **Не запускайте сервер одновременно через SSH и админ панель** - это вызовет конфликт
2. **Используйте один способ запуска** - либо через админ панель, либо через SSH
3. **Если процесс убивается (Killed)**, проверьте логи: `tail -f server/api.log`
4. **Если не хватает памяти**, уменьшите лимит в скриптах с `512` до `256`
