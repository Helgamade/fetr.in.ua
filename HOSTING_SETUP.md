# Инструкция по настройке хостинга для API

## Шаг 1: Включить проксирование трафика

1. Перейдите в **"Настройки сайта"** → **"Основные настройки"**
2. В разделе **"Веб-сервер"** найдите опцию **"Проксирование трафика (настройки)"**
3. **Включите** эту опцию (поставьте галочку)
4. Нажмите **"Сохранить"**

## Шаг 2: Настроить запуск Node.js приложения

1. Перейдите в **"Настройки веб-приложения"**
2. Заполните следующие поля:
   - **Каталог запуска:** `/home/idesig02/fetr.in.ua/www/`
   - **Команда запуска:** `node server/index.js`
3. Нажмите **"Сохранить"**

## Шаг 3: Проверить package.json

Убедитесь, что в `package.json` есть скрипт для запуска сервера:

```json
{
  "scripts": {
    "server": "node server/index.js"
  }
}
```

## Шаг 4: Проверить переменные окружения

Убедитесь, что файл `server/.env` содержит правильные настройки:

```env
DB_HOST=idesig02.mysql.tools
DB_USER=idesig02_fetrinua
DB_PASSWORD=7%j-7EyzF8
DB_NAME=idesig02_fetrinua
DB_PORT=3306
PORT=3001
NODE_ENV=production
CORS_ORIGIN=https://fetr.in.ua
```

## Шаг 5: Настроить переменные окружения для Node.js (если нужно)

В настройках Node.js можно использовать переменные окружения:
- `process.env.PORT` - порт (будет установлен автоматически)
- `process.env.HOST` - хост (будет установлен автоматически)

## После настройки:

1. Приложение должно автоматически запуститься
2. API будет доступен через проксирование на основном домене
3. Фронтенд должен использовать относительный путь `/api` для запросов

## Проверка работы:

После настройки проверьте:
- Статус приложения должен быть "Работает" (зеленый)
- API должен быть доступен по адресу `https://fetr.in.ua/api/health`
- Фронтенд должен загружать данные без ошибок
